Варіант №2: Соціальна мережа (70 балів)
Обирайте цей варіант якщо ви дуже впевнені в своєму рівні знань і хочете завдання з зірочкою. 
1. Створення додатку (10 балів):
	o Розробити серверний додаток на базі Express або NestJS (4 бали).
	o Використати PostgreSQL як основну базу даних (3 бали).
	o Налаштувати Swagger для документування API (3 бали).
2. Сутності (10 балів):
	o Створити таблицю users для зберігання основної інформації про користувачів (5 балів).
	o Створити таблицю posts для зберігання інформації про пости користувачів (5 балів).
3. CRUD операції (15 балів):
	o Для users:
		• POST /user для створення користувача з хешованим паролем (bcrypt) (5 балів).
	o Для posts:
		• POST /posts для створення нового посту для авторизованого користувача (5 балів).
		• GET /posts/
		• для отримання всіх постів користувача з кешуванням в Redis (5 балів).
4. JWT авторизація (10 балів):
	o Реалізувати JWT авторизацію з токеном, який має термін дії (10 балів).
	o Токен приходить на запит get-token, з логіном і паролем юзера. 
5. Кешування через Redis (5 балів):
	o Реалізувати кешування результатів GET-запитів до /posts/
	o через Redis (5 балів).
6. Валідація та помилки (5 балів):
	o Реалізувати 401 Unauthorized для запитів без валідного токена (3 бали).
	o Реалізувати 403 Forbidden при відсутності необхідних параметрів (2 бали).
7. Docker та Docker Compose (10 балів):
	o Налаштувати Docker для контейнеризації додатку (5 балів).
	o Налаштувати Docker Compose для одночасного запуску серверу, PostgreSQL та Redis (5 балів).
8. Тестування (5 балів):
	o Написати юніт-тести для CRUD операцій, авторизації та кешування (3 бали).
	o Додати інтеграційні тести для перевірки роботи додатку (2 бали).
